# CL-2026-099 — Sample claim for testing

**Selection mode:** REQUEST  
**Domain:** Build infrastructure  
**Scope:** Workbench repo validation

---

## Claim

The workbench build pipeline correctly transforms Markdown files into blueprint-compliant HTML pages.

## Predictions

If true, any `.md` file placed in a topic subfolder should appear as a styled page at the corresponding path under `_site/`, using the T(h)reehouse +EC design tokens.

## Constraints

- CD blueprint v1.7.0 governs all visual output.
- GitHub Actions must complete without external dependencies beyond `raw.githubusercontent.com`.
- No JavaScript frameworks; pure Python build.

## Comparison

Build output is inspectable at the GitHub Pages URL. Visual compliance can be checked against the cd-rules demonstrator.

## Classification

**UNDERDETERMINED**

**Discriminant Condition:** Run the build in GitHub Actions and visually compare output against the cd-rules demonstrator page.  
**Feasibility:** High — requires only a push to `main`.
