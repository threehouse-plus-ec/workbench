# Mobile-first open science infrastructure

## Opening

The assumption that scholarly writing requires a laptop is a friction point that restricts when and where ideas can be developed. A git-based workflow operable from a phone removes that constraint.

## Breakwater summary

See CL-2026-099 for the initial validation of the build pipeline.

## Synthesis

By combining GitHub's mobile editing capabilities with a zero-dependency Python build script, the entire cycle from draft to published page can run without touching a desktop machine. The corporate design tokens travel with the build, ensuring visual consistency without manual intervention.

## Open questions

- How should image assets be handled in a mobile-only workflow?
